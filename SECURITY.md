# Security Policy

## Supported versions

Only the latest minor version is supported with security fixes.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

1. Contact maintainers privately with reproduction details.
2. Include affected versions, impact, and proof of concept if available.
3. We will acknowledge receipt and follow up with remediation steps.

## Disclosure policy

- We aim to provide an initial response within 72 hours.
- We will coordinate disclosure once a fix is available.
